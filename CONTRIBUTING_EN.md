Thank you for your interest in contributing! This file provides guidelines and instructions to help you contribute effectively.

## Code of Conduct

Be respectful, provide constructive feedback, and focus on what is best for the project.

## Getting Started

1. Fork the repository on GitHub.
2. Clone your fork:

```bash
git clone https://github.com/your-username/backend_login_page.git
cd backend_login_page
```

3. Create a feature branch:

```bash
git checkout -b feature/your-feature
```

4. Implement changes and add tests when applicable.

5. Run tests locally:

```bash
mvn test
```

6. Follow Conventional Commits for commit messages.

## Commit Types

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation changes
- `test:` tests
- `chore:` maintenance tasks
- `ci:` CI/CD changes

## Pull Request

Open a pull request against the `main` branch with a clear description, test notes and changelog when applicable.

## Templates

Use the issue and PR templates when provided.
